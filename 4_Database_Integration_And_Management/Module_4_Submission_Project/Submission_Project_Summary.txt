Basic SQL queries
    For this project I created four basic SQL queries using Copilot. The first retrieves ProductName, Category, Price, and StockLevel from the Products table using a SELECT statement. The second filters products by a specific category using a WHERE clause (e.g. WHERE Category = 'Electronics'). The third filters products with low stock using WHERE StockLevel < 20. The fourth sorts all products by price from lowest to highest using ORDER BY Price ASC.

Complex SQL queries
    I created multi-table JOINs connecting Products, Sales, Stores, and Suppliers to display ProductName, SaleDate, StoreLocation, and UnitsSold. I used aggregate functions SUM, AVG, MAX, and COUNT with GROUP BY to calculate total revenue per product, supplier performance, and category trends. I also wrote nested subqueries to identify products with above-average sales volumes. Using Copilot, I debugged three errors: ambiguous column references in JOIN clauses fixed with explicit table aliases, a misplaced WHERE in a correlated subquery fixed with a derived-table JOIN, and missing GROUP BY columns causing syntax errors. For optimization, Copilot recommended adding six indexes on frequently queried columns such as Category, ProductID, StoreID, and SaleDate, and replacing IN subqueries with pre-aggregated derived-table JOINs to improve execution speed.
